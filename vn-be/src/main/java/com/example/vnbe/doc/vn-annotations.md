# Vn Entity ‚Äî Annotations Guide

Based on the `vn` database table structure, here's what annotations to use for each field and why.

---

## Class-Level Annotations

### `@Entity`
Required ‚Äî tells JPA this class maps to a database table.

### `@Table(name = "vn")`
The class is named `Vn` which JPA would map to `vn` by default, but it's good practice to be explicit so there's no ambiguity.

### `@Data` (Lombok)
Generates getters, setters, `toString`, `equals`, `hashCode`. Avoids boilerplate.

### `@NoArgsConstructor` (Lombok)
JPA requires a no-arg constructor to instantiate entities via reflection.

### `@AllArgsConstructor` (Lombok)
Convenience for creating instances with all fields set (useful in tests and builders).

## Field-Level Annotations

### `id` ‚Äî `text NOT NULL` (Primary Key)
```java
@Id
@Column(name = "id", nullable = false)
private String id;
```
- **`@Id`** ‚Äî marks this as the primary key
- **No `@GeneratedValue`** ‚Äî the `id` is `text` with no default/sequence, meaning it's assigned externally (e.g., `v17`), not auto-generated by the database

### `image` ‚Äî `text NULL` (FK ‚Üí images)
```java
@Column(name = "image")
private String image;
```
- **No `nullable = false`** ‚Äî column is nullable, and JPA defaults to nullable, so no constraint needed
- Mapped as a plain `String` since we're just storing the foreign key value. If you later want a relationship to an `Image` entity, replace with `@ManyToOne` + `@JoinColumn`

### `c_image` ‚Äî `text NULL`
```java
@Column(name = "c_image")
private String cImage;
```
- **`@Column(name = "c_image")`** ‚Äî required because the Java field name `cImage` (camelCase) doesn't match the column name `c_image` (snake_case)

### `olang` ‚Äî `language NOT NULL` (Custom PostgreSQL Enum)
```java
@Column(name = "olang", nullable = false, columnDefinition = "language")
@Enumerated(EnumType.STRING)
private Language olang;
```
- **`@Enumerated(EnumType.STRING)`** ‚Äî stores the enum as its name string (e.g., `"ja"`, `"en"`), matching the PostgreSQL enum values
- **`columnDefinition = "language"`** ‚Äî tells Hibernate the actual DB type is the custom `language` enum, not a standard `varchar`
- **`nullable = false`** ‚Äî column is NOT NULL
- You need to create a Java `Language` enum with values matching the DB enum (`ar`, `be`, `bg`, ... `zh-Hant`)
- **Note:** You'll also need a custom Hibernate type or `@Type` annotation to properly handle the PostgreSQL enum ‚Üî Java enum conversion

### `c_votecount` ‚Äî `integer NOT NULL`
```java
@Column(name = "c_votecount", nullable = false)
private Integer cVotecount;
```
- **`@Column(name = "c_votecount")`** ‚Äî camelCase ‚âÝ snake_case, explicit mapping needed
- **`nullable = false`** ‚Äî enforces the NOT NULL constraint

### `c_rating` ‚Äî `smallint NULL`
```java
@Column(name = "c_rating")
private Short cRating;
```
- **`Short`** ‚Äî maps to `smallint` in PostgreSQL
- Nullable column ‚Üí use `Short` (wrapper) not `short` (primitive), because primitives can't be null

### `c_average` ‚Äî `smallint NULL`
```java
@Column(name = "c_average")
private Short cAverage;
```
- Same reasoning as `c_rating` ‚Äî nullable `smallint` ‚Üí `Short` wrapper type

### `c_length` ‚Äî `smallint NULL`
```java
@Column(name = "c_length")
private Short cLength;
```
- Same pattern ‚Äî nullable `smallint`

### `c_lengthnum` ‚Äî `smallint NOT NULL`
```java
@Column(name = "c_lengthnum", nullable = false)
private Short cLengthnum;
```
- **`nullable = false`** ‚Äî NOT NULL column
- Could use primitive `short` since it's never null, but `Short` is fine for consistency

### `length` ‚Äî `smallint NOT NULL`
```java
@Column(name = "length", nullable = false)
private Short length;
```
- `length` is a reserved word in some SQL dialects but PostgreSQL handles it fine. If issues arise, use `@Column(name = "\"length\"")`

### `devstatus` ‚Äî `smallint NOT NULL`
```java
@Column(name = "devstatus", nullable = false)
private Short devstatus;
```
- Straightforward NOT NULL smallint mapping

### `alias` ‚Äî `text NOT NULL`
```java
@Column(name = "alias", nullable = false, columnDefinition = "TEXT")
private String alias;
```
- **`columnDefinition = "TEXT"`** ‚Äî optional but explicit; without it Hibernate may default to `varchar(255)` which doesn't match the `text` type

### `description` ‚Äî `text NOT NULL`
```java
@Column(name = "description", nullable = false, columnDefinition = "TEXT")
private String description;
```
- Same reasoning as `alias` ‚Äî use `TEXT` column definition to match the actual DB type

## Key Takeaways

1. **No `@GeneratedValue`** ‚Äî the `id` is assigned externally, not auto-generated
2. **Use `Short` not `short`** for nullable `smallint` columns ‚Äî primitives can't be null
3. **Use `@Column(name = "...")` everywhere** snake_case columns don't match camelCase fields ‚Äî or configure Hibernate's `PhysicalNamingStrategy` to do it globally
4. **Custom enum handling** ‚Äî `olang` uses a PostgreSQL enum type, which needs `columnDefinition` and potentially a custom Hibernate type
5. **Use `columnDefinition = "TEXT"`** for text columns to prevent Hibernate defaulting to `varchar(255)`
